PROJECT: ChainBridge SCF Command Simulator — Enterprise Edition
TYPE: WebGL-accelerated interactive supply chain network simulator
ARCHITECTURE: React + react-force-graph-2d (WebGL) + XState + GSAP + deck.gl

This is a production-grade, GPU-accelerated supply chain finance 
simulator. Not a dashboard. Not a chart app. A live command center 
that a J.P. Morgan coverage head could use in an actual operations 
room. Every visual choice must reflect this.

──────────────────────────────────────────
CORE ARCHITECTURE — NON-NEGOTIABLE
──────────────────────────────────────────

PRIMARY GRAPH ENGINE:
  react-force-graph-2d
  
  This is WebGL-accelerated via HTML5 Canvas. NOT SVG. NOT React Flow.
  It handles particle flow along edges natively via 
  linkDirectionalParticles and linkDirectionalParticleSpeed.
  Nodes are rendered via custom canvasObject callbacks — full 
  canvas 2D API control per node.
  This is the library used in production network intelligence tools.

SIMULATION STATE MACHINE:
  XState v5
  
  The simulation is a proper finite state machine, not ad-hoc 
  useState logic. States:
    idle → running → event_processing → cascading → 
    recovering → stable
  
  Each stress event is an XState event that triggers a transition.
  ChainBridge intervention is a separate machine that runs in 
  parallel, consuming signals and producing intervention events.
  This architecture means the simulation behaves predictably and 
  can be stepped backward/forward (undo/redo of simulation ticks).

ANIMATION ENGINE:
  GSAP 3 (GreenSock) — for all timeline-based animations
  Framer Motion — for panel slide-ins and modal transitions only
  requestAnimationFrame — for the simulation tick loop
  
  GSAP is not optional. Every stress cascade, every recovery wave, 
  every health meter update must go through a GSAP timeline.
  This produces professional easing curves that Framer Motion 
  and CSS transitions cannot match.

GLOBAL STATE:
  Zustand with Immer middleware
  All simulation state in one store, all mutations immutable.

CHARTS INSIDE PANELS:
  Nivo (not Recharts) — for the sparklines, bar charts, and 
  arc meters in side panels.
  Nivo renders with D3 under the hood and has motion support 
  out of the box.

GEOGRAPHIC BASE LAYER:
  Render a simplified SVG APAC region map as the canvas background.
  Use a dark political boundary outline of APAC with:
    Singapore, India, Vietnam, Hong Kong labeled
  Scale is not geographic precision — it is schematic.
  Position supplier nodes loosely by corridor geography.
  Anchor nodes float above the geographic layer in the upper half.
  
  The effect: the simulator looks like it is operating on a real 
  region, not an abstract graph.

SIMULATION ENGINE IN WEB WORKER:
  The tick calculation runs in a Web Worker (worker.js) so the 
  main thread is never blocked. The main thread only handles 
  render and interaction. Worker posts state diffs on each tick.
  This ensures the graph never stutters during stress cascades.

──────────────────────────────────────────
EDGE DESIGN — PARTICLE FLOW
──────────────────────────────────────────

All edges use react-force-graph-2d's built-in:
  linkDirectionalParticles
  linkDirectionalParticleWidth
  linkDirectionalParticleSpeed
  linkDirectionalParticleColor

HEALTHY EDGE:
  linkColor: rgba(0, 255, 135, 0.25)
  linkWidth: proportional to weight_wij, range 1–5px
  linkDirectionalParticles: 4
  linkDirectionalParticleWidth: 2
  linkDirectionalParticleSpeed: 0.004
  linkDirectionalParticleColor: #00FF87
  
  Effect: green particles flow FROM supplier TOWARD anchor, 
          representing invoice financing volume moving up the chain.

AT-RISK EDGE:
  linkColor: rgba(255,184,0, 0.3)
  linkDirectionalParticles: 2
  linkDirectionalParticleSpeed: 0.002
  linkDirectionalParticleColor: #FFB800
  Particles slow down — visually implies stress, reduced flow.

BROKEN / CRITICAL EDGE:
  linkColor: rgba(255,68,68,0.5)
  linkDirectionalParticles: 0
  linkDashPattern: [4, 4] — custom rendering via canvasLinkObject
  No particles. The broken flow is communicated by absence.

DISPLACED EDGE (to Competitor):
  linkColor: rgba(107,114,128,0.35)
  linkDirectionalParticles: 3
  linkDirectionalParticleSpeed: 0.006  (faster — volume actively leaving)
  linkDirectionalParticleColor: #9CA3AF
  Particle direction: TOWARD competitor node

MULTI-ANCHOR EDGES:
  Suppliers connected to 3+ anchors: edges rendered as curved 
  bezier links (use react-force-graph curvature property, 0.2–0.4)
  This prevents edge crossings from obscuring the concentration 
  pattern and makes the network structure legible.

GSAP STRESS CASCADE ANIMATION
──────────────────────────────────────────

When a stress event fires (any scenario trigger):

1. GSAP timeline starts immediately.

2. Step 1 (0ms): The triggering node pulses.
   GSAP to: {scale: 1.4, duration: 0.2, yoyo: true, repeat: 1}
   Achieved by modifying the node radius in the render function.

3. Step 2 (200ms): Edges from that node transition to at-risk.
   GSAP staggerTo: each connected edge's color property, 
   stagger: 0.08s per edge, duration: 0.3s
   
4. Step 3 (400ms + stagger): Connected anchor nodes receive 
   a ripple — their outer ring briefly flares to amber.
   GSAP to: ringOpacity 1 → amber glow → back, duration 0.6s

5. Step 4 (Cross-anchor contagion only): 
   For each other supplier connected to the same anchors:
   GSAP staggerTo with stagger 0.05s, amber ripple on each node.
   Total cascade takes 800–1200ms depending on network depth.
   This is the money animation. The cascade should look like 
   a visible shockwave moving through the supply chain.

6. Recovery (ChainBridge Active):
   Reverse timeline: green particles re-appear on edges one by one.
   GSAP staggerTo: edges recover with stagger 0.12s.
   Node status dots flip green with a brief white flash (0.1s).
   Programme Health Meters animate up with GSAP to: progress values.
   
   Total recovery animation: 3–4 seconds, feels earned.

──────────────────────────────────────────
DATA LOADING + INITIALIZATION
──────────────────────────────────────────

Files in /public/data/:
  gap1_programme_health.csv
  gap2_onboarding_entity_resolution.csv
  gap3_utilisation_classification.csv
  gap4_cross_anchor_network.csv
  gap5_displacement_detection.csv

On app load:
  1. Load all CSVs with PapaParse async
  2. Build Zustand store:
     - nodes: merged supplier profile from all 5 CSVs
     - edges: from gap4 with weight_wij
     - anchors: 5 anchor nodes from gap4 unique anchor_ids
     - competitorNode: manually initialized
  3. Initialize XState simulation machine in idle state
  4. Pass graphData (nodes + links) to react-force-graph-2d

react-force-graph-2d config:
  dagMode: null (force-directed, not DAG)
  d3AlphaDecay: 0.02 (slow physics settle = smooth layout)
  d3VelocityDecay: 0.3
  warmupTicks: 100 (pre-compute layout before render)
  cooldownTicks: Infinity (keep physics running for drag)
  nodeCanvasObject: custom function (full node rendering)
  linkCanvasObjectMode: "after" for edge labels on hover
  linkDirectionalParticles: dynamic per edge state
  enableZoomInteraction: true
  enablePanInteraction: true
  minZoom: 0.3, maxZoom: 8

──────────────────────────────────────────
PERFORMANCE REQUIREMENTS
──────────────────────────────────────────
- Graph must run at 60fps with 200 nodes and 408 edges.
  react-force-graph-2d handles this — do not add DOM nodes 
  inside the canvas area. All node rendering via canvasObject.
  
- Simulation tick (Web Worker) must not block render thread.
  Worker communicates via postMessage with state diffs only 
  (not full state on each tick).

- GSAP animations must use will-change: transform where applicable.

- Nivo charts inside panels must use isInteractive: false 
  on sparklines to prevent unnecessary re-renders.

──────────────────────────────────────────
WHAT IS EXPLICITLY FORBIDDEN
──────────────────────────────────────────
- No Three.js 3D. This is 2D. The depth comes from layering 
  and glow, not from a Z-axis.
- No React Flow. react-force-graph-2d only for the network.
- No Recharts. Nivo only.
- No placeholder lorem ipsum data. All data from CSVs.
- No tutorial overlays or onboarding tooltips.
- No mobile responsive layout. This is a desktop/projector tool.
- No dark/light mode toggle.
- No sounds or haptics.
- No AI chat panel.
- No company logos from the internet (use text initials only).
- Do not add any feature not specified in this document.
